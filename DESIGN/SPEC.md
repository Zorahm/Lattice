# Lattice UI — спека

Десктоп-клиент (Windows), Svelte + Tauri 2. Принцип: главный экран как у
Hamachi — два поля и кнопка. Вся сложность в настройках.

## Экран 1 — вход
- Поле «Название сети», поле «Пароль» (со скрытием).
- Подсказка: «Тот, кто введёт такие же название и пароль, попадёт в эту же сеть».
- Кнопка «Подключиться» (крупная, главная). Иконка настроек в углу.
- Чекбокс «Запомнить» (опц.).
- Кнопка неактивна, пока поля пусты. При нажатии → «Подключение…».

Название+пароль → backend (KDF→ключ→network-id). Фронт ключ не трогает.

## Экран 2 — в сети
- Шапка: название сети, статус, свой overlay-IP.
- Список пиров, каждый: индикатор связи (● зелёный=напрямую, ● жёлтый=relay,
  ○ серый=офлайн), имя, overlay-IP, опц. пинг.
- Легенда цветов. Кнопка «Отключиться» (неакцентная).
- Пусто: «Поделитесь названием и паролем, чтобы пригласить других».

## Экран 3 — настройки
Секции, у каждого поля рабочий дефолт (пустые настройки = работает из коробки):
- **Сеть:** подсеть (10.66.0.0/24), назначение IP, MTU (~1380).
- **Сервер:** адрес координации, STUN-серверы.
- **Соединение:** разрешить relay (вкл), порт (авто), keepalive (под «расширенные»).
- **Приложение:** автозапуск, трей, язык.
- **Диагностика:** тип NAT, внешний endpoint, копировать лог.

## Поведение
- Трей: окно сворачивается в трей, не выходит.
- Ошибки человеческим языком, не errno:
  - нет прав админа → «Запустите от имени администратора — нужно для адаптера».
  - нет TAP-драйвера → «Не найден сетевой драйвер. Установить?».
  - сервер недоступен → «Не удаётся связаться с сервером. Проверьте интернет/адрес».
  - relay вместо p2p → не ошибка, жёлтый индикатор + tooltip.
- Реконнект: статус «Переподключение…», авто-ретрай, без модалок на каждый разрыв.

## НЕ на главных экранах
Подсеть/порт/MTU/STUN/сервер — только в настройках. Термины NAT/punching/relay —
только цвет+tooltip, детали в Диагностике. Одна основная кнопка за раз.

## Тон
Простой русский, без жаргона. Кнопки — глаголы («Подключиться»). Сообщения —
что произошло + что делать.
